# Day 26: Passive Voice + Relative Clauses cho IT

> 🎯 **Mục tiêu:** Dùng bị động và mệnh đề quan hệ — phổ biến trong docs, email, và thảo luận kỹ thuật.

---

## Phiên 1: Passive Voice (25 phút)

### 1.1 Passive trong IT — rất phổ biến!
**Cấu trúc:** S + be + V3 (past participle)

| Active | Passive |
|---|---|
| We **deployed** the app. | The app **was deployed**. |
| The team **fixed** the bug. | The bug **was fixed**. |
| They **will release** v2.0. | v2.0 **will be released**. |
| Someone **has updated** the docs. | The docs **have been updated**. |
| We **are testing** the feature. | The feature **is being tested**. |

### 1.2 Khi nào dùng Passive?
- Không quan trọng AI làm: "The data **is encrypted** at rest."
- Quy trình: "The code **is reviewed** before merging."
- Docs/email: "The release **has been postponed**."

### 1.3 Ví dụ IT thực tế
1. "The database **is backed up** every 6 hours."
2. "All requests **are logged** for monitoring."
3. "The vulnerability **was patched** in the latest update."
4. "User input **must be validated** before processing."
5. "The feature **will be rolled out** next sprint."

## Phiên 2: Relative Clauses (25 phút)

### 2.1 Relative pronouns
- **who** (người): "The developer **who** fixed the bug..."
- **which/that** (vật): "The API **that** handles authentication..."
- **where** (nơi): "The server **where** the app is hosted..."

### 2.2 Ví dụ IT
1. "The function **that returns** a promise needs error handling."
2. "The server **which is running** on port 3000 crashed."
3. "The developer **who wrote** this code left the company."
4. "The database **where** we store user data is PostgreSQL."
5. "The bug **that caused** the crash has been fixed."

### 2.3 Bài tập
Nối 2 câu bằng relative clause:
- "React is a library. It is used for building UIs."
  → "React is a library **that is used for** building UIs."
- "John is a developer. He works on the backend."
  → "John is a developer **who works on** the backend."

## Phiên 3: Từ Vựng — Documentation (15 phút)

| # | Từ | Nghĩa |
|---|---|---|
| 1 | document | tài liệu |
| 2 | specification | đặc tả |
| 3 | guideline | hướng dẫn |
| 4 | reference | tham khảo |
| 5 | maintain | duy trì |
| 6 | deprecated | không dùng nữa |
| 7 | legacy | cũ, kế thừa |
| 8 | compatible | tương thích |
| 9 | release notes | ghi chú phát hành |
| 10 | changelog | nhật ký thay đổi |

Nhập Anki + ôn.

## Phiên 4: Nói + Viết (15 phút)
Viết 1 đoạn release notes bằng passive voice:
> "Version 2.1 has been released. The login bug was fixed. A new dashboard feature was added. The API response time was improved by 30%. The old endpoint has been deprecated and will be removed in v3.0."

Đọc to → ghi âm.
