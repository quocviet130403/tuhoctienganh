# Day 9: Phụ Âm /ʃ/ vs /ʒ/ vs /tʃ/ vs /dʒ/

> 🎯 **Mục tiêu:** Phát âm đúng 4 âm "sh/zh/ch/j", phân biệt chúng khi nói và nghe.

---

## Phiên 1: Phát Âm (30 phút)

### 1.1 Âm /ʃ/ — "sh" (7 phút)
- **Cách phát âm:** Môi hơi chúm tròn, lưỡi nâng lên nhưng không chạm, thổi hơi → KHÔNG rung cổ

| Từ | Phiên âm | Nghĩa |
|---|---|---|
| show | /ʃoʊ/ | hiển thị |
| share | /ʃer/ | chia sẻ |
| push | /pʊʃ/ | đẩy |
| cache | /kæʃ/ | bộ nhớ đệm |
| crash | /kræʃ/ | sập |
| function | /ˈfʌŋk**ʃ**ən/ | hàm |
| condition | /kənˈdɪ**ʃ**ən/ | điều kiện |
| application | /ˌæplɪˈkeɪ**ʃ**ən/ | ứng dụng |

### 1.2 Âm /ʒ/ — "zh" (7 phút)
- **Cách phát âm:** Giống /ʃ/ nhưng CÓ rung cổ họng (ít gặp hơn trong tiếng Anh)

| Từ | Phiên âm | Nghĩa |
|---|---|---|
| measure | /ˈmeʒər/ | đo lường |
| version | /ˈvɜːrʒən/ | phiên bản |
| vision | /ˈvɪʒən/ | tầm nhìn |
| usually | /ˈjuːʒuəli/ | thường |
| decision | /dɪˈsɪʒən/ | quyết định |
| revision | /rɪˈvɪʒən/ | sửa đổi |

### 1.3 Âm /tʃ/ — "ch" (7 phút)
- **Cách phát âm:** Đầu lưỡi chạm vòm miệng → bật ra nhanh (giống "ch" tiếng Việt)

| Từ | Phiên âm | Nghĩa |
|---|---|---|
| check | /tʃek/ | kiểm tra |
| change | /tʃeɪndʒ/ | thay đổi |
| choose | /tʃuːz/ | chọn |
| chart | /tʃɑːrt/ | biểu đồ |
| branch | /bræntʃ/ | nhánh |
| match | /mætʃ/ | khớp |
| fetch | /fetʃ/ | tìm nạp |
| feature | /ˈfiːtʃər/ | tính năng |

### 1.4 Âm /dʒ/ — "j" (7 phút)
- **Cách phát âm:** Giống /tʃ/ nhưng CÓ rung cổ họng

| Từ | Phiên âm | Nghĩa |
|---|---|---|
| join | /dʒɔɪn/ | tham gia |
| page | /peɪdʒ/ | trang |
| change | /tʃeɪndʒ/ | thay đổi |
| large | /lɑːrdʒ/ | lớn |
| manage | /ˈmænɪdʒ/ | quản lý |
| image | /ˈɪmɪdʒ/ | hình ảnh |
| JavaScript | /ˈdʒɑːvəskrɪpt/ | JavaScript |
| JSON | /ˈdʒeɪsən/ | JSON |

### 1.5 So sánh 4 âm

| Vô thanh | Hữu thanh |
|---|---|
| /ʃ/ share (thổi) | /ʒ/ version (rung) |
| /tʃ/ check (bật) | /dʒ/ join (bật + rung) |

**Bài tập:** Đọc: share → version → check → join → lặp lại 5 lần.

## Phiên 4: Tập Nói (10 phút)

### Mô tả quy trình testing
Ghi âm bản thân giải thích:

> "In software development, testing is very important. First, I write unit tests to check each function. Then I run integration tests to make sure the features work together. If any test fails, I need to fix the code and check for regression. I want to achieve high test coverage."

Đọc mẫu 2 lần → nói không nhìn → ghi âm → nghe lại.
