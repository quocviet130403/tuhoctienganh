## Phiên 1: Linking Words (25 phút)

### 1.1 Nối ý — Sequence
| Từ | Ví dụ |
|---|---|
| First / Firstly | **First**, I cloned the repo. |
| Then / Next | **Then**, I created a new branch. |
| After that | **After that**, I wrote the code. |
| Finally | **Finally**, I pushed the changes. |

### 1.2 Nguyên nhân — Reason
| Từ | Ví dụ |
|---|---|
| because | The test failed **because** of a typo. |
| since | **Since** the server is slow, we added caching. |
| due to | The crash was **due to** a memory leak. |

### 1.3 Đối lập — Contrast
| Từ | Ví dụ |
|---|---|
| however | It works locally. **However**, it fails in production. |
| although | **Although** the code is correct, it's not efficient. |
| but | The feature is done, **but** it needs testing. |
| on the other hand | SQL is consistent. **On the other hand**, NoSQL is flexible. |

### 1.4 Bổ sung — Addition
| Từ | Ví dụ |
|---|---|
| also | I **also** added error handling. |
| in addition | **In addition**, we need to update the docs. |
| moreover | **Moreover**, the API needs rate limiting. |

## Phiên 2: Viết Email IT (30 phút)

### 2.1 Cấu trúc email cơ bản
```
Subject: [Clear, specific subject]

Hi [Name],

[Opening - purpose of email]
[Body - details]
[Closing - next steps]

Best regards,
[Your name]
```

### 2.2 Email mẫu 1 — Báo cáo bug
```
Subject: Bug Report: Login page crashes on mobile

Hi Team,

I found a bug on the login page. When users try to log in
on mobile devices, the page crashes with a null pointer error.

Steps to reproduce:
1. Open the app on a mobile browser
2. Enter username and password
3. Click the login button

The expected behavior is a successful login, but the actual
result is a blank screen with a console error.

I've attached the error logs. Could someone look into this?

Best regards,
[Name]
```

### 2.3 Email mẫu 2 — Hỏi ý kiến
```
Subject: Question about database migration approach

Hi [Name],

I'm working on the database migration task. I have two
options and would like your opinion:

Option A: Migrate all data at once during downtime
Option B: Gradual migration with zero downtime

Option A is simpler but requires 2 hours of downtime.
Option B is more complex, however, it won't affect users.

Which approach would you recommend?

Thanks,
[Name]
```

### 2.4 Bài tập
Viết 1 email: chọn 1 trong các chủ đề:
- Xin review PR
- Báo cáo tiến độ
- Hỏi kỹ thuật

## Phiên 4: Nói (10 phút)

> "**First**, I noticed the API was slow. **Then**, I checked the database queries. I found that one query was missing an index. **Because of** this, it was scanning the entire table. **After** I added the index, the response time improved. **However**, we also need to add caching **in addition to** the index."
