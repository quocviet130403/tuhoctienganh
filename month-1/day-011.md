# Day 11: Phụ Âm Cuối + Âm /s/ /z/ /ɪz/

> 🎯 **Mục tiêu:** Phát âm đúng phụ âm cuối (ending consonants) — lỗi cực kỳ phổ biến của người Việt. Học cách phát âm đuôi -s/-es.

---

## Phiên 1: Phát Âm (30 phút)

### 1.1 Vì sao phụ âm cuối quan trọng? (3 phút)
- Người Việt hay **nuốt âm cuối** vì tiếng Việt không có phụ âm cuối mạnh
- Ví dụ: "test" → nói thành "tes" | "start" → nói thành "star"
- **Hậu quả:** Người nghe không hiểu hoặc hiểu sai nghĩa

### 1.2 Luyện phụ âm cuối cơ bản (12 phút)

#### Âm /t/ cuối
| Từ | Phải nghe thấy /t/ |
|---|---|
| get /ɡet/ | "t" rõ ràng |
| set /set/ | "t" rõ ràng |
| test /test/ | "t" cuối |
| script /skrɪpt/ | "pt" cuối |
| start /stɑːrt/ | "t" cuối |
| output /ˈaʊtpʊt/ | "t" cuối |

#### Âm /d/ cuối
| Từ | Phải nghe thấy /d/ |
|---|---|
| code /koʊd/ | "d" rõ ràng |
| load /loʊd/ | "d" cuối |
| build /bɪld/ | "ld" cuối |
| failed /feɪld/ | "ld" cuối |
| fixed /fɪkst/ | "kst" cuối |

#### Âm /k/ cuối
| Từ |
|---|
| back /bæk/ |
| check /tʃek/ |
| stack /stæk/ |
| block /blɒk/ |
| link /lɪŋk/ |

#### Âm /p/ cuối
| Từ |
|---|
| app /æp/ |
| stop /stɒp/ |
| map /mæp/ |
| step /step/ |
| drop /drɒp/ |

**Bài tập:** Đọc từng từ → TẤN CÔNG vào âm cuối, nói HƠI PHÓNG ĐẠI → dần về bình thường.

### 1.3 Đuôi -s / -es: Ba cách phát âm (15 phút)

#### Quy tắc:

| Phát âm | Khi nào | Ví dụ |
|---|---|---|
| **/s/** | Sau âm vô thanh: /p/ /t/ /k/ /f/ /θ/ | stops, tests, stacks |
| **/z/** | Sau âm hữu thanh và nguyên âm | runs, files, codes, arrays |
| **/ɪz/** | Sau /s/ /z/ /ʃ/ /tʃ/ /dʒ/ | cases, pages, changes |

#### Bài tập từng nhóm:

**Nhóm /s/:**
- stops /stɒps/, gets /ɡets/, stacks /stæks/, steps /steps/, tests /tests/

**Nhóm /z/:**
- runs /rʌnz/, codes /koʊdz/, files /faɪlz/, servers /ˈsɜːrvərz/, tools /tuːlz/

**Nhóm /ɪz/:**
- cases /ˈkeɪsɪz/, pages /ˈpeɪdʒɪz/, changes /ˈtʃeɪndʒɪz/, fixes /ˈfɪksɪz/, pushes /ˈpʊʃɪz/

**Bài tập:** Đọc 3 nhóm → ghi âm → kiểm tra bạn có phát âm khác nhau không?

## Phiên 4: Tập Nói (10 phút)

### Đọc đoạn Agile workflow
> "In our **sprints**, we pick **tasks** from the **backlog**. Each **task** has a **deadline** and **priority**. We have daily **standups** to **check** progress. At the **end** of the **sprint**, we **test** all **features** and get **feedback** from **stakeholders**."

Đọc chậm, chú ý TỪNG âm cuối → ghi âm → nghe lại.
