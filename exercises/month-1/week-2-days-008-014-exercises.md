**Ex 3: Từ vựng Networking — Điền từ**
1. The _______ blocks incoming traffic. (firewall)
2. HTTP is a _______. (protocol)
3. The _______ is too high. (latency)
4. Check the _______ settings. (DNS)
5. _______ the data before sending. (Encrypt)

### 🎧 Listening Text

```
Let me explain how computer networks work.

A network is a group of computers connected together. They can share data and resources. The internet is the largest network in the world.

When you send data over a network, it is broken into small pieces called packets. Each packet travels through routers to reach its destination. A router is a device that directs traffic on the network.

A firewall is a security system that monitors incoming and outgoing traffic. It blocks harmful data and protects your network from attacks.

Every computer on a network has an IP address. Think of it as a home address for your computer. The DNS system translates website names like "google dot com" into IP addresses.

Latency is the time it takes for data to travel from one point to another. High latency means slow connections. Bandwidth is the amount of data that can be sent at once. More bandwidth means faster transfers.

To keep data safe, we use encryption. Encryption converts readable data into a coded form. Only someone with the right key can decrypt and read the data. SSL certificates are used to encrypt data between a browser and a server.
```

**Ex 2: Từ vựng Testing — True/False**
1. A unit test checks the entire application. (F — checks individual units)
2. Test coverage measures how much code is tested. (T)
3. A mock replaces a real component with a fake one. (T)
4. Regression testing checks for new features. (F — checks old features still work)
5. An assertion verifies an expected result. (T)

### 🎧 Listening Text

```
Software testing is a critical part of development. Let me explain the different types of tests.

Unit tests check individual functions or methods. For example, if you have a function that adds two numbers, a unit test would verify that giving it three and five returns eight. Unit tests are fast and should cover as much code as possible.

Integration tests check how different parts of your application work together. For example, testing that your API correctly saves data to the database. These tests are slower than unit tests but catch problems that unit tests miss.

End-to-end tests simulate real user behavior. They open a browser, click buttons, fill forms, and verify the results. These tests are the slowest but give you the most confidence that your application works correctly.

We measure test quality using code coverage. Coverage tells you what percentage of your code is tested. While one hundred percent coverage is ideal, most teams aim for seventy to eighty percent.

When we find a bug, we write a test that reproduces the bug first. Then we fix the code so the test passes. This way, the same bug will never happen again. This approach is called test-driven development, or TDD.
```

**Ex 3: Security vocabulary — Matching**
| Từ | Nghĩa |
|---|---|
| 1. authentication | a. lỗ hổng |
| 2. authorization | b. xác thực |
| 3. vulnerability | c. mã thông báo |
| 4. token | d. phân quyền |
| 5. encryption | e. mã hóa |

**Đáp án:** 1-b, 2-d, 3-a, 4-c, 5-e

### 🎧 Listening Text

```
Security is one of the most important aspects of software development. A single vulnerability can expose millions of users' data.

Authentication is the process of verifying who you are. When you enter your username and password, the system checks if they match. If they do, you are authenticated. Modern applications also use two-factor authentication, which adds an extra layer of security by sending a code to your phone.

Authorization is different from authentication. Authorization determines what you are allowed to do. For example, a regular user can view their profile, but only an admin can delete user accounts.

Tokens are commonly used for both authentication and authorization. When you log in, the server generates a token — usually a JSON Web Token, or JWT. Your browser stores this token and sends it with every request. The server checks the token to verify your identity without asking for your password again.

Encryption protects data by converting it into a coded form. There are two types: encryption at rest, which protects stored data, and encryption in transit, which protects data being sent over the network.

Always hash passwords before storing them in the database. Never store passwords in plain text. Use a strong hashing algorithm like bcrypt and add a salt for extra security.
```

**Ex 2: Agile vocabulary — Điền từ**
1. We plan our work in two-week _______. (sprints)
2. Add the task to the _______. (backlog)
3. We have a daily _______ at 10 AM. (standup)
4. What's the _______ for this story? (estimate)
5. The _______ is Friday at 5 PM. (deadline)

### 🎧 Listening Text

```
Agile is a method of software development that focuses on delivering work in small, frequent pieces called iterations or sprints.

In a typical Agile team, we use a sprint cycle of two weeks. At the beginning of each sprint, we have a planning meeting. The team looks at the product backlog, which is a list of all the tasks and features that need to be built. We pick the most important items and estimate how much effort each one requires.

Every day, we have a standup meeting. It lasts about fifteen minutes. Each team member answers three questions: What did I do yesterday? What will I do today? Am I blocked by anything?

At the end of the sprint, we have two meetings. First, a sprint review where we demonstrate what we built to the stakeholders. They give us feedback, and we adjust our priorities for the next sprint.

Then we have a retrospective. We discuss what went well, what did not go well, and what we can improve. This continuous improvement cycle is at the heart of Agile.

Estimation is important but difficult. Some teams use story points instead of hours. Story points measure the relative complexity of a task. A task with two story points is roughly twice as complex as a task with one story point.
```

### 📝 Exercises

| /t/ | /d/ | /ɪd/ |
|---|---|---|
| ? | ? | ? |

**Đáp án:** /t/: fixed, pushed, crashed | /d/: deployed, called, merged | /ɪd/: started, needed, created, updated

**Ex 2: DevOps — True or False**
1. CI/CD stands for Continuous Integration and Continuous Delivery. (T)
2. Staging is the same as production. (F)
3. A rollback returns the app to a previous version. (T)
4. Docker containers are like virtual machines. (Partially T — lighter than VMs)
5. Monitoring helps detect problems early. (T)

### 🎧 Listening Text

```
DevOps is a set of practices that combines software development and IT operations. The goal is to deliver software faster and more reliably.

Continuous Integration, or CI, means that developers merge their code changes into the main branch frequently — usually several times a day. Each merge triggers an automated build and test process. If the tests fail, the developer is notified immediately and can fix the problem before it affects others.

Continuous Delivery, or CD, extends CI by automatically deploying code to a staging environment after the tests pass. The staging environment is a copy of production where the team can test the application in realistic conditions. Once the team is satisfied, they can deploy to production with a single click.

Docker containers make deployment easier. A container packages your application with all its dependencies, so it runs the same way on any machine. You build a Docker image once, and you can run it anywhere.

Monitoring is essential in production. Tools like Prometheus and Grafana help you track metrics like response time, error rate, and CPU usage. When something goes wrong, alerts notify the team so they can respond quickly.

If a deployment causes problems, we can do a rollback — reverting to the previous version of the application. This is why it is important to keep your deployments small and frequent.
```

### 📝 Day 14 Exercises — Ôn Tập Tuần 2

**Grand Test: chọn phát âm đúng**

2. "version" — /ˈvɜːrʒən/ hay /ˈvɜːrʃən/? → /ˈvɜːrʒən/
3. "checked" — /tʃekt/ hay /tʃekɪd/? → /tʃekt/

5. "deployed" — /dɪˈplɔɪt/ hay /dɪˈplɔɪd/? → /dɪˈplɔɪd/

### 🎧 Listening Text Day 13-14

```
Cloud computing has changed the way we build and deploy software. Instead of buying and maintaining physical servers, companies can rent computing resources from cloud providers like Amazon Web Services, Google Cloud Platform, or Microsoft Azure.

There are three main types of cloud services. Infrastructure as a Service, or IaaS, gives you virtual machines, storage, and networking. Platform as a Service, or PaaS, provides tools and services for building applications without managing the underlying infrastructure. Software as a Service, or SaaS, delivers complete applications over the internet, like Gmail or Slack.

One of the biggest advantages of cloud computing is scalability. If your application suddenly gets more traffic, the cloud can automatically add more servers to handle the load. When traffic decreases, it scales back down, so you only pay for what you use.

Cloud providers also offer high availability. Your data is stored in multiple locations, so if one server fails, another takes over. This ensures that your application stays online even during hardware failures.

To manage cloud resources, many teams use Infrastructure as Code tools like Terraform or CloudFormation. These tools let you define your infrastructure in configuration files, making it easy to create, update, and destroy resources automatically.
```
