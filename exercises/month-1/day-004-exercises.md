## 🎧 LISTENING TEXT — Day 4

### Listening 4.1: How Websites Work (Chậm)

```
Let me explain how websites work in simple terms.

When you type a website address in your browser and press Enter, your computer sends a request to a server. The server is a powerful computer that stores the website files.

The server receives your request and sends back a response. This response contains the web page — the text, images, and code that make up what you see on your screen.

The browser reads this code and shows you the web page. This all happens in less than one second.

Every website has a domain name, like "google dot com." Behind the domain name is an IP address, which is a number that identifies the server. The Domain Name System, or DNS, translates the domain name into an IP address.

When you fill out a login form and click the submit button, the browser sends your username and password to the server. The server checks the database to see if your information is correct. If it matches, the server sends back a success response, and you are logged in.
```

### Listening 4.2: Client-Server Architecture (Bình thường)

```
In web development, we often talk about the client-server architecture. Let me break it down for you.

The client is the user's device — usually a browser like Chrome or Firefox. The server is a remote computer that processes requests and sends back data.

When a user opens a website, the client sends an HTTP request to the server. The request includes information about what the user wants — for example, loading a page or submitting a form.

The server processes the request. It might query a database to get the right information. Then it creates a response and sends it back to the client. The response includes a status code. A status code of 200 means everything is OK. A status code of 404 means the page was not found. A 500 error means something went wrong on the server.

Modern web applications often separate the frontend and backend. The frontend runs on the client and handles the user interface. The backend runs on the server and handles data processing, business logic, and database operations.

The frontend and backend communicate through APIs — Application Programming Interfaces. The frontend sends API requests, and the backend returns data, usually in JSON format.
```

### Listening 4.3: Dictation

```
The browser sends a request to the server.
The server returns a response with data.
Check the status code of the response.
The login form sends the username and password.
The database stores all user information.
Deploy the application to the production server.
The client and server communicate through an API.
Register a domain name for your website.
```
