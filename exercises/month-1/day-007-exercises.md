## 🎧 LISTENING TEXT — Day 7

### Listening 7.1: Week 1 Recap — Everything Together (Chậm)

```
Let me review what we have learned in the first week.

We started by learning about computers. A computer has hardware and software. The hardware includes the screen, keyboard, and mouse. The software includes the programs and applications we use.

Then we learned about basic actions. We click, type, open, close, save, copy, paste, search, install, and run. These are the actions we perform every day as developers.

Next, we studied programming basics. We learned about variables, functions, loops, arrays, and booleans. A variable stores a value. A function is a block of reusable code. A loop repeats an action. An array holds a collection of items. A boolean is either true or false.

We also learned about web development. A browser sends requests to a server. The server processes the request and sends a response. The data is stored in a database. When the application is ready, we deploy it to production.

After that, we studied Git and version control. We clone repositories, create branches, commit changes, push code, and create pull requests. Git helps teams work together on the same codebase without conflicts.

Finally, we learned about data structures — arrays, stacks, queues, hash maps, and trees. Each data structure is good for different use cases.

Great work! You have learned over ninety vocabulary words and the foundations of English pronunciation. Keep going!
```

### Listening 7.2: A Developer's Journey (Bình thường)

```
My name is Sarah and I am a full-stack developer. Let me tell you about how I became a developer.

I started learning programming three years ago. At first, I only knew how to write simple code — variables, functions, and loops. I used YouTube videos and online courses to learn.

After a few months, I built my first website. It was a simple page with HTML and CSS. Then I learned JavaScript and started making interactive web applications. I used React for the frontend and Node dot JS for the backend.

I learned Git early on, and that was one of the best decisions I made. Being able to track my changes and collaborate with others made everything easier. I create branches for every feature, commit often with clear messages, and always open pull requests for code review.

Understanding data structures helped me write better code. When I need fast lookups, I use a hash map. When I need to process tasks in order, I use a queue. Knowing which data structure to choose makes my programs faster and more efficient.

Today, I work at a tech company. I deploy code to production every week. I write tests, review code, and mentor junior developers. I still learn something new every day.

My advice to you is: practice consistently. Write code every day, even if it is just for thirty minutes. Read documentation in English. Watch tech videos in English. And never be afraid to ask questions.
```
