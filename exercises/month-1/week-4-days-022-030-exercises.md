## Day 22-23: Present Simple/Continuous + Past Simple

### 📝 Exercises

**Ex 1: Chọn thì đúng**
1. I _______ (write / am writing) code every day. → **write** (thói quen)
2. Right now, I _______ (fix / am fixing) a bug. → **am fixing** (đang làm)
3. Yesterday, I _______ (deploy / deployed) the app. → **deployed** (hôm qua)
4. The server _______ (runs / is running) on port 3000. → **runs** (sự thật)
5. We _______ (use / are using) a new framework this month. → **are using** (tạm thời)
6. Last week, she _______ (built / builds) the authentication module. → **built** (tuần trước)
7. He usually _______ (commits / committed) code before lunch. → **commits** (thói quen)
8. I _______ (didn't test / don't test) the feature yet. → **didn't test** (chưa)

### 🎧 Listening Text — A Developer's Typical Day

```
Let me describe a typical day in my life as a software developer.

I usually wake up at seven thirty in the morning. After breakfast, I start work at nine. The first thing I do is check my email and Slack messages. Then I join the daily standup meeting where I tell my team what I did yesterday, what I plan to do today, and whether I have any blockers.

After the standup, I start coding. Right now, I am working on a new payment feature. I write code in TypeScript and test it in my local environment. I commit my changes frequently with clear messages.

Yesterday was a busy day. I fixed three bugs in the morning. One of them was a tricky issue with the database connection. I found that the connection pool was running out of connections during peak hours. I increased the pool size and added better error handling.

In the afternoon, I reviewed two pull requests from my colleagues. I suggested some improvements and approved one of them. Then I wrote unit tests for the feature I completed last week.

Before the end of the day, I pushed my code and updated the task board. I also sent a progress report to my team lead.

Today, I am continuing work on the payment feature. I need to integrate it with the third-party payment gateway. I am also attending a design meeting at two o'clock to discuss the next sprint.
```

## Day 24-25: Present Perfect + Conditionals + Modal Verbs

### 📝 Exercises

**Ex 1: Present Perfect vs Past Simple**
Chọn thì đúng:
1. I _______ (used / have used) React for 3 years. → **have used** (kéo dài đến hiện tại)
2. I _______ (built / have built) an app last year. → **built** (thời gian cụ thể)
3. She _______ (never worked / has never worked) with Docker. → **has never worked** (kinh nghiệm)
4. We _______ (deployed / have deployed) 50 releases this year. → **have deployed** (đến hiện tại)
5. He _______ (fixed / has fixed) that bug yesterday. → **fixed** ("yesterday" = past simple)

**Ex 2: Conditionals — Hoàn thành câu**
1. If we add caching, the app _______ (will be) faster. (Type 1)
2. If I had more time, I _______ (would refactor) the code. (Type 2)
3. If the tests _______ (pass), I'll deploy. (Type 1)
4. If we _______ (used) TypeScript, we would have fewer bugs. (Type 2)
5. If you _______ (don't) back up, you will lose data. (Type 1)

**Ex 3: Modal verbs — Chọn modal phù hợp**
*(can, should, would, might, must, need to)*
1. You _______ validate user input. (must — bắt buộc)
2. We _______ use Redis for caching. (could — đề xuất)
3. This _______ cause a memory leak. (might — có thể)
4. I _______ recommend TypeScript. (would — khuyến nghị)
5. You _______ write unit tests. (should — nên)

### 🎧 Listening Text — Interview Practice

```
Now let me show you how to talk about your experience in a job interview.

When an interviewer asks "Tell me about your experience," they want to know what you have done throughout your career, not just one specific event.

Use the Present Perfect tense for accumulated experience: "I have worked as a developer for five years." "I have built several full-stack applications." "I have managed a team of four developers."

Use the Past Simple when talking about specific projects or events: "Last year, I built a real-time chat application." "In two thousand twenty-three, I led a database migration project."

For discussing solutions, use conditionals and modal verbs. "If we used a microservices architecture, we could scale each service independently." "We should consider caching frequently accessed data." "This approach might improve response time by fifty percent."

Here is an example answer: "I have over five years of experience in backend development. I have worked with Node dot JS, Python, and Go. Currently, I am working at a fintech company where I have built several microservices. Last year, I led a project to migrate our monolithic application to microservices, which reduced deployment time by seventy percent. If I were to do it again, I would start with better documentation."

Notice how this answer uses Present Perfect for overall experience, Past Simple for specific achievements, Present Continuous for current work, and conditionals for reflection.
```

## Day 26-27: Passive Voice + Linking Words + Email

### 📝 Exercises

**Ex 1: Active → Passive**
1. We deployed the app. → The app _______ _______ . (was deployed)
2. Someone fixed the bug. → The bug _______ _______ . (was fixed)
3. They have updated the docs. → The docs _______ _______ _______. (have been updated)
4. We are testing the feature. → The feature _______ _______ _______. (is being tested)
5. They will release v2.0 tomorrow. → v2.0 _______ _______ _______ tomorrow. (will be released)

**Ex 2: Linking words — Điền từ nối**
*(because, however, first, then, finally, in addition, although, due to)*
1. _______, I cloned the repo. _______, I created a branch. (First, Then)
2. The test failed _______ a typo in the config. (because of / due to)
3. The code works locally. _______, it fails in production. (However)
4. _______ the code is correct, it's not efficient. (Although)
5. We added caching. _______, we optimized the queries. (In addition)

**Ex 3: Viết email**
Viết email báo cáo tiến độ cho team lead (dùng passive + linking words):
```
Subject: Weekly Progress Update

Hi [Name],

[Opening: tóm tắt tuần]
[Body: chi tiết — dùng passive: "The feature was completed..."]
[Issues: "However, we encountered..."]
[Next steps: "Next week, we will..."]

Best regards,
[Your name]
```

### 🎧 Listening Text — Professional Email & Communication

```
Writing professional emails is an essential skill for any IT professional. Let me share some tips and examples.

First, always write a clear subject line. "Bug Report: Login page crashes on mobile" is much better than "Problem" or "Help."

Start with a greeting. "Hi Team" or "Hi Sarah" is appropriate for most work emails. Avoid "Dear Sir or Madam" unless it is a very formal situation.

Get to the point quickly. The first sentence should state the purpose of your email. For example: "I'd like to update you on the progress of the payment feature."

Use linking words to organize your ideas. "First, I completed the database schema. Then, I implemented the API endpoints. However, I encountered an issue with the payment gateway integration."

The passive voice is common in professional writing. Instead of "I finished the feature," you might write "The feature has been completed." Instead of "Someone broke the build," write "The build was broken." This sounds more professional and focuses on the action rather than the person.

End with clear next steps. "I will have the fix ready by Friday" or "Could you please review the pull request by end of day?"

Close with a professional sign-off: "Best regards," "Thanks," or "Cheers" for informal communication.

Remember: keep emails short and to the point. If your email is longer than three paragraphs, consider scheduling a meeting instead.
```
