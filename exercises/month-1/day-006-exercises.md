## 🎧 LISTENING TEXT — Day 6

### Listening 6.1: Data Structures Explained (Chậm)

```
Data structures are ways to organize and store data in a computer. Choosing the right data structure is very important because it affects how fast your program runs.

Let me explain some common data structures.

An array is a list of items stored in order. Each item has a position number called an index. The first item has index zero, the second has index one, and so on. Arrays are good for accessing items quickly by their index, but adding or removing items in the middle is slow.

A stack is like a pile of books. You can only add a book to the top or remove a book from the top. This is called "Last In, First Out," or LIFO. In programming, we use stacks for things like the undo button in a text editor.

A queue is like a line of people waiting. The first person in line is the first to be served. This is called "First In, First Out," or FIFO. We use queues for things like processing tasks in order.

A hash map, also called a dictionary, stores data as key-value pairs. For example, you could store a person's name as the key and their email as the value. Hash maps are very fast for looking up values by their key.

A tree is a structure where each item, called a node, can have children. The top node is called the root. Trees are used in many applications, like file systems and search algorithms.
```

### Listening 6.2: Choosing the Right Data Structure (Bình thường)

```
One of the most important skills for a developer is knowing which data structure to use for a given problem. Let me give you some examples.

If you need to store a list of items and access them by position, use an array. For example, storing the top ten scores in a game.

If you need to process items in the order they arrive, use a queue. For example, handling customer support tickets — the first ticket submitted should be handled first.

If you need to undo actions, use a stack. Push each action onto the stack. When the user clicks undo, pop the last action from the stack and reverse it.

If you need fast lookups by a key, use a hash map. For example, checking if a username already exists in your system. Instead of searching through every user, you can look it up directly by the username key, which takes constant time.

If your data has a hierarchical structure, like categories and subcategories, use a tree. For example, a file system where folders contain files and other folders.

Remember, there is no single best data structure. The right choice depends on your specific use case. Think about what operations you need to perform most often — inserting, searching, sorting, or deleting — and choose accordingly.
```

### Listening 6.3: Dictation

```
An array stores items in order with an index.
A stack follows the Last In First Out principle.
A queue processes items in the order they arrive.
A hash map provides fast lookups by key.
A tree has nodes with parent and child relationships.
Choose the right data structure for your problem.
Sorting an array takes O of n log n time.
Insert a new element at the end of the list.
```
