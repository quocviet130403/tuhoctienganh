## Day 45: Daily Standup English

> 🎯 Nói trôi chảy trong cuộc họp standup hàng ngày.

### Phiên 1: Standup framework (30 phút)

**3 câu hỏi:**
1. What did you do yesterday?
2. What will you do today?
3. Any blockers?

**Mẫu:**
> "Yesterday, I **finished** the user authentication module and **submitted** a pull request. I also **fixed** a minor bug in the login page. **Today**, I'm **going to** work on the payment integration. I need to set up the Stripe webhook. I don't have any **blockers** at the moment, but I **might** need help with the webhook testing later."

**Cụm từ hữu ích:**
- "I'm on track with..."
- "I'm a bit behind on... because..."
- "I'm blocked by... I need [person] to..."
- "I'll carry over [task] from yesterday."
- "I'm waiting on [person/thing] to proceed."

### Bài tập: Viết standup hôm nay → nói to → ghi âm. Làm mỗi ngày từ giờ!

## Day 47: Phone/Video Call English

> 🎯 Giao tiếp tự tin qua điện thoại/video call.

### Phiên 1: Call phrases (30 phút)

**Opening:**
- "Hi [name], can you hear me okay?"
- "Sorry, my mic was on mute."
- "Let me share my screen."
- "Can everyone see my screen?"

**During call:**
- "Could you repeat that? The connection was bad."
- "Sorry, you're breaking up."
- "Let me check that and get back to you."
- "Does anyone have any questions?"
- "I'll drop the link in the chat."

**Closing:**
- "Let me summarize what we discussed..."
- "I'll send the meeting notes after this."
- "Thanks everyone. Talk to you later!"

## Day 49: Giving & Receiving Feedback

> 🎯 Cho và nhận feedback trong code review, meetings.

### Phiên 1: Giving constructive feedback (25 phút)

**Framework: Situation → Behavior → Impact → Suggestion**

**Code review feedback:**
- "I noticed that this function is quite long. **Would** it be better to split it into smaller functions for readability?"
- "Great work on the error handling! **One suggestion** — maybe we could add a retry mechanism here?"
- "This approach works, **but** I think using a Map instead of an Object would be more efficient for this use case."
- "I **like** how you structured the tests. **However**, I think we should add an edge case for empty input."

**Receiving feedback:**
- "That's a great point. I'll update that."
- "I see what you mean. Let me refactor that part."
- "Thanks for catching that! I'll fix it right away."
- "Could you help me understand why that approach is better?"

### Phiên 2: Practice (25 phút) — Review sample code → write feedback
