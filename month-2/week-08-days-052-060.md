## Day 52: Câu Điều Kiện Nâng Cao + Subjunctive

> 🎯 Dùng conditionals phức tạp hơn để thảo luận kỹ thuật sâu.

### Phiên 1: Advanced Conditionals (30 phút)

**Type 3 — Giả định quá khứ (nếu lúc đó đã...):**
- "If we **had used** TypeScript from the start, we **would have caught** these bugs earlier."
- "If I **had known** about this library, I **wouldn't have** built it from scratch."

**Mixed conditionals:**
- "If we **had** designed the database properly, we **wouldn't be** having performance issues now."

**Unless / As long as / Provided that:**
- "The deployment will succeed **unless** there are failing tests."
- "You can push directly **as long as** it's a hotfix."
- "I'll approve the PR **provided that** you add unit tests."

### Bài tập: Viết 5 câu Type 3 về kinh nghiệm thực tế.

## Day 53: Discussion & Meeting Skills

> 🎯 Dẫn dắt và tham gia thảo luận kỹ thuật.

### Leading a discussion:
- "Let's start by discussing..."
- "I'd like to get everyone's thoughts on..."
- "Let's move on to the next topic."
- "To summarize, we've decided to..."
- "Are there any objections?"

### Participating:
- "I'd like to add something..."
- "Building on what [name] said..."
- "Going back to the earlier point about..."
- "Can we table this and come back to it later?"
- "I have a quick suggestion..."

### Bài tập: Thực hành dẫn dắt 1 cuộc thảo luận giả lập (10 phút) → ghi âm.

## Day 54: Writing — PR Descriptions & Commit Messages

> 🎯 Viết PR description và commit messages chuẩn tiếng Anh.

### Commit message conventions:
```
feat: add user authentication module
fix: resolve null pointer in payment service
docs: update API documentation
refactor: simplify database query logic
test: add unit tests for auth middleware
chore: update dependencies
```

### PR description template:
```
## What does this PR do?
Adds user authentication using JWT tokens.

## Why is this change needed?
We need to secure API endpoints and track user sessions.

## How was this tested?
- Unit tests for token generation and validation
- Integration tests for login/logout flow
- Manual testing on staging environment

## Screenshots (if applicable)
[Screenshots here]

## Checklist
- [x] Tests pass
- [x] Code reviewed
- [x] Documentation updated
```

### Bài tập: Viết 5 commit messages + 1 PR description cho code thực tế.

## Day 56: Negotiation & Estimation

> 🎯 Thảo luận timeline, ước lượng, và đàm phán trong IT.

### Estimation phrases:
- "I estimate this will take about 3 days."
- "It's hard to give an exact estimate, but roughly..."
- "In the best case, 2 days. Worst case, a week."
- "I'd need to investigate further before giving an estimate."

### Pushing back (politely):
- "I'm concerned that this timeline is too aggressive."
- "We might need to descope some features to meet the deadline."
- "Could we extend the deadline by a few days?"
- "If we cut [feature], we can deliver by [date]."

### Bài tập: Role-play estimation meeting → ghi âm.

## Day 57: Presentation Skills — Technical Demo

> 🎯 Trình bày demo sản phẩm/tính năng bằng tiếng Anh.

### Demo structure:
1. **Context:** "I'd like to show you the new [feature]."
2. **Problem:** "Previously, users had to..."
3. **Demo:** "Let me walk you through it. First... Then... As you can see..."
4. **Impact:** "This reduces [metric] by X%."
5. **Questions:** "Any questions about what I just showed?"

### Bài tập: Demo 1 tính năng (thật hoặc giả) → ghi hình 5 phút.
