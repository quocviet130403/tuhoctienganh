## Day 38: Past Tenses in Context — Kể Dự Án Chi Tiết

> 🎯 Kể chi tiết về 1 dự án sử dụng Past Simple + Past Continuous + Past Perfect.

### Phiên 1: Ngữ pháp nâng cao (30 phút)

#### Past Continuous — Đang làm (quá khứ)
- "I **was working** on the API when the server crashed."
- "While the team **was testing**, I **was fixing** bugs."

#### Past Perfect — Hành động xảy ra trước 1 hành động khác
- "I **had already deployed** the app before the bug was found."
- "We **had finished** testing when the client changed the requirements."

#### Kết hợp 3 thì:
> "Last year, I **was working** on a fintech project. We **had been** using a monolithic architecture, but we **decided** to migrate to microservices. I **built** the payment service while my colleague **was developing** the user service. Before we **launched**, we **had run** extensive load tests."

### Bài tập: Viết 1 đoạn 150 từ kể dự án → ghi âm đọc to.

## Day 39: Describing Technology Stack

> 🎯 Giải thích tech stack của bạn một cách rõ ràng.

### Phiên 1: Framework mô tả tech stack (30 phút)

**Cấu trúc:**
1. **Frontend:** "For the frontend, we use/used [framework] because [reason]."
2. **Backend:** "The backend is built with [language/framework]."
3. **Database:** "We store data in [database] because [reason]."
4. **Infrastructure:** "Everything runs on [cloud] with [tools]."
5. **Why:** "We chose this stack because [reasons]."

**Mẫu:**
> "The frontend is built with React and TypeScript. We chose React because of its large ecosystem and component-based architecture. For the backend, we use Node.js with Express. Our database is PostgreSQL for structured data and Redis for caching. Everything is deployed on AWS using Docker containers and managed by Kubernetes."

### Bài tập: Mô tả tech stack dự án của bạn → ghi âm.

## Day 40: Mô Tả Quy Trình Làm Việc (Workflow)

> 🎯 Giải thích development workflow bằng tiếng Anh.

### Phiên 1: Development workflow (30 phút)

**Sequence words:** First → Then → After that → Next → Finally

**Mẫu:**
> "Here's our typical development workflow. **First**, we receive requirements from the product team. **Then**, we break them down into user stories and estimate the effort. **After that**, developers pick up tasks from the backlog. We create a feature branch, write code, and submit a pull request. **Next**, another developer reviews the code. If there are issues, we address the feedback. **Once** everything is approved, we merge into the main branch. The CI/CD pipeline **then** automatically builds and deploys to staging. **Finally**, after QA testing, we deploy to production."

### Bài tập: Mô tả workflow thực tế của bạn → ghi âm.

## Day 41: Describing Technical Challenges

> 🎯 Kể về thử thách kỹ thuật và cách vượt qua — skill phỏng vấn quan trọng.

### Phiên 1: Challenge narrative framework (30 phút)

**Cấu trúc (Pre-STAR):**
1. **Context:** "We were building a [system] and..."
2. **Challenge:** "The main challenge was..."
3. **Approach:** "I decided to... / My approach was to..."
4. **Actions:** "First, I... Then, I... After research, I..."
5. **Result:** "As a result, we... / The outcome was..."
6. **Learning:** "I learned that..."

**Mẫu:**
> "We were building a real-time notification system. The main challenge was handling 100,000 concurrent connections. Initially, we used polling, but it was too slow. I researched WebSocket alternatives and decided to use Socket.IO with Redis adapter. I implemented connection pooling and message queuing. As a result, we could handle 150,000 concurrent connections with 99.9% uptime. I learned the importance of choosing the right architecture pattern early."

### Bài tập: Viết 2 challenge stories → ghi âm.

## Day 42: Comparing Technologies

> 🎯 So sánh 2 công nghệ một cách chuyên nghiệp.

### Phiên 1: Comparison phrases (30 phút)

**Hơn:**
- "X is **faster** than Y."
- "X **offers more** flexibility **than** Y."
- "X is **better suited for** large-scale applications."

**Kém:**
- "X is **not as** scalable **as** Y."
- "X **has fewer** community resources **than** Y."

**Bằng:**
- "X is **as** reliable **as** Y."
- "Both X and Y **support**..."

**Preference:**
- "I **prefer** X **over** Y **because**..."
- "I **would choose** X **for** this use case."
- "It **depends on** the requirements."

### Bài tập: So sánh React vs Vue, hoặc SQL vs NoSQL → ghi âm 3 phút.

## Day 43: Explaining Code / Architecture

> 🎯 Giải thích code và kiến trúc hệ thống bằng tiếng Anh.

### Phiên 1: Code explanation (30 phút)

**Framework:**
1. **Purpose:** "This function/class is responsible for..."
2. **Input:** "It takes [parameters] as input."
3. **Process:** "First, it checks... Then, it..."
4. **Output:** "It returns..."
5. **Why:** "I designed it this way because..."

**Mẫu:**
> "This middleware function handles authentication. It takes the request object as input. First, it extracts the JWT token from the Authorization header. Then, it verifies the token using our secret key. If the token is valid, it attaches the user data to the request and calls next(). Otherwise, it returns a 401 Unauthorized error. I chose JWT because it's stateless and works well with our microservices architecture."

### Bài tập: Chọn 1 function bạn đã viết → giải thích → ghi âm.
