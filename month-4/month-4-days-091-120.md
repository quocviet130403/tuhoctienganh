## Tuần 13 (Days 91–97): Daily Standup & Meeting Mastery

### Day 91: Advanced Standup — Saying More with Less

**Standup anti-patterns (tránh):**
- ❌ Quá chi tiết: "I opened the file then I wrote a function that..."
- ❌ Quá chung: "I worked on stuff."
- ✅ Vừa đủ: "I completed the authentication middleware. Today I'll integrate it with the user service."

**Cụm từ nâng cao:**
- "I'm making good progress on..."
- "I hit a roadblock with... and I'm exploring alternatives."
- "I need to sync with [person] about..."
- "I underestimated the complexity of... so I'll need another day."
- "This is at risk of slipping. We might want to discuss priorities."

### Day 92: Sprint Planning Conversations
**Estimation phrases:**
- "I'd estimate this at [X] story points."
- "This seems like a [size] effort."
- "There's some uncertainty around... which could affect the estimate."
- "Can we break this into smaller tasks?"
- "What are the acceptance criteria?"

### Day 93: Sprint Review & Demo Skills
- Present completed work

- Demo application features live

### Day 94: Retrospective Facilitation
- "What went well this sprint?"

### Day 95: 1-on-1 Meeting with Manager

- "I'd like to discuss my career growth."
- "I'm interested in working on..."

### Day 96: Cross-Team Communication

## Tuần 14 (Days 98–104): Code Review Communication Mastery

### Day 98: Writing Code Review Comments

**Frameworks for comments:**
| Type | Prefix | Example |
|---|---|---|
| Must fix | `[BLOCKER]` | "[BLOCKER] This query is vulnerable to SQL injection." |
| Suggestion | `[SUGGESTION]` | "[SUGGESTION] Consider using a Set instead of Array for O(1) lookups." |
| Question | `[QUESTION]` | "[QUESTION] Why did you choose this approach over...?" |
| Nitpick | `[NIT]` | "[NIT] Typo in variable name on line 42." |
| Praise | `[PRAISE]` | "[PRAISE] Great error handling here!" |

### Day 99: Verbal Code Review
- Walking through code changes in a meeting
- "Let me walk you through the changes."
- "The key change here is..."
- "I'd like your opinion on this approach."

### Day 100: Responding to Code Review Feedback
- "Good catch! I'll fix that."
- "I chose this approach because... but I'm open to alternatives."
- "I see your point. Let me refactor that."
- "Could you elaborate on that suggestion?"
