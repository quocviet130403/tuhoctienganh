## Tuần 9 (Days 61–67): Từ Vựng Phát Triển Phần Mềm

### Day 61: Software Development Lifecycle (SDLC)

**Từ vựng SDLC:**
| Từ | Phiên âm | Nghĩa | Câu ví dụ |
|---|---|---|---|
| requirement | /rɪˈkwaɪərmənt/ | yêu cầu | Gather the requirements. |
| analysis | /əˈnæləsɪs/ | phân tích | Do a requirements analysis. |
| design | /dɪˈzaɪn/ | thiết kế | Design the system architecture. |
| implement | /ˈɪmplɪment/ | triển khai | Implement the feature. |
| deploy | /dɪˈplɔɪ/ | triển khai lên server | Deploy to production. |
| maintain | /meɪnˈteɪn/ | bảo trì | Maintain the codebase. |
| iterate | /ˈɪtəreɪt/ | lặp lại | Iterate on the design. |
| prototype | /ˈproʊtətaɪp/ | nguyên mẫu | Build a prototype first. |
| feasibility | /ˌfiːzəˈbɪlɪti/ | tính khả thi | Study the feasibility. |
| specification | /ˌspesɪfɪˈkeɪʃən/ | đặc tả | Write the specification. |

**Bài nói:** Mô tả SDLC ở công ty bạn bằng tiếng Anh (3 phút) → ghi âm.

### Day 62: Code Quality & Best Practices

**Từ vựng:**
| Từ | Nghĩa | Câu ví dụ |
|---|---|---|
| clean code | mã sạch | Write clean code. |
| code smell | mã có vấn đề | This function has a code smell. |

| DRY | Don't Repeat Yourself | Apply the DRY principle. |

| abstraction | trừu tượng hóa | Use the right level of abstraction. |

### Day 65: Agile Ceremonies Deep Dive

**Mẫu câu cho mỗi ceremony:**

**Sprint Planning:**
- "Based on the velocity, I think we can take on..."
- "This story seems too large. Should we break it down?"
- "I estimate this at [X] story points because..."

**Sprint Review:**
- "During this sprint, we completed..."
- "Let me demo the new feature..."
- "We didn't finish [task] because [reason]."

**Retrospective:**
- "What went well: [item]"
- "What could be improved: [item]"
- "Action item: [item], assigned to [person]"

**Bài nói:** Giả lập 1 sprint review (5 phút) → ghi hình.
